# 🕊️ Ластівка

**Проактивний privacy-preserving захист прав дитини** — крос-реєстрова система раннього виявлення порушень прав дитини в Україні.

Жодне порушення не доводить один реєстр — доводить **перетин** даних кількох. Ластівка збирає дитину з силосованих державних реєстрів (не зливаючи персональні дані), виявляє порушення на перетині сигналів, ранжує за терміновістю й віддає кейсворкеру пояснену чергу.

> Демонстрація на **синтетичних** даних, які (1) структурно повторюють **точні схеми реальних реєстрів** (поля з законодавства) і (2) відтворюють **реальну епідеміологію** дитячих даних України (справжні base rates і кореляції). Жодних справжніх персональних даних.

---

## Що нового (фаза 2): реалізм

- **Точні схеми реєстрів** — поля з законів/постанов/офіційних форм (див. `docs/REGISTRY_SCHEMAS.md`), не «на око».
- **Реальна епідеміологія** — base rates і кореляції з дослідження (50 агентів: Держстат, UNICEF, OHCHR, Мінсоц, НСЗУ, МОН, ILO). Більшість дітей **без порушень** (~77% чисті); рідкісне лишається рідкісним; фактори **корельовані** (коморбідність). Параметри — `config/epidemiology.yaml`, провенанс — `docs/genparams.json`.
- **Гетерогенність** — кожна дитина = унікальний сампл латентного контексту → різні життєві шляхи, не клони.

---

## Чотири стовпи
1. **Інтеграція даних** — зібрати життєві події з реєстрів у профіль дитини.
2. **Виявлення ризиків** — тривожні сигнали й критичні відхилення до кризи.
3. **Цілеспрямоване реагування** — ранжований сигнал відповідальному спеціалісту.
4. **Управлінська панель** — моніторинг і підтримка рішень.

## Архітектура

| # | Модуль | Що робить |
|---|---|---|
| 1 | `realmodel` (+ `entity`) | реалістична популяція: демографія → латентні контекст-фактори (корельовані) → УМОВНЕ призначення порушень (P = base × Π множників) → траєкторія в часі |
| 2 | `emitters` + `storage` | кожен реєстр НЕЗАЛЕЖНО пише у власну SQLite-базу (силос) з **точними реальними полями** + шум + `access_level` |
| 3 | `matching` | зібрати дитину з силосів: точно по УНЗР, fuzzy (`rapidfuzz`) по ПІБ+дата де УНЗР нема (id_map — різні реальні назви полів) |
| 4 | `detection` | формули порушень (перетин реєстрів) + change-point/dropout |
| 5 | `scoring` | тріаж: urgency-score → рівень черги T0/T1/T2 з поясненням |
| 6 | `privacy` | PPRL (Bloom-filter/Dice) + PSI-булеан для даних Рівня 1 |
| 8 | `validation` | precision/recall vs ground truth (+ MLflow) |

**Метод:** темпоральна агентна симуляція (стиль Synthea): причина = латентний фактор/шок, симптом = крос-реєстровий лог-патерн виникає сам. Детектор бачить лише силоси, не god-view.

## Запуск

```bash
pip install -r requirements.txt

python scripts/run_generation.py -n 5000      # силосовані реєстри (точні схеми)
python scripts/run_pipeline.py --no-mlflow    # match → detect → score → validate
streamlit run app/dashboard.py                # дашборд → http://localhost:8501
uvicorn app.api:app --port 8000               # (опц.) API → /docs
```
Масштаб: `-n 50000`. Параметри: `config/config.yaml` (генерація), `config/epidemiology.yaml` (епідеміологія/ваги ризиків), `config/scoring.yaml` (ваги тріажу).

---

## Метрики (5 000 дітей)

- **Реалізм:** ~77% дітей чисті · бідність ~64% · ВПО ~15% · сироти ~0.9% · депортовані ~0.3% · інвалідність ~2.9% — у межах реальних оцінок.
- **Матчинг:** ~96.6% дітей зібрано з силосів у 1 сутність.
- **Детекція:** precision ~0.92 · recall ~0.85 · F1 ~0.88 (vs синтетичний ground truth).
- **PPRL (без plaintext):** precision ~1.00 · recall ~0.95.

## Етика
Decision **support**, не decision making: система пріоритезує й **пояснює**, рішення ухвалює спеціаліст. Виявляємо **об'єктивну депривацію прав**, а не «прогнозуємо злочинність батьків» — це обходить головну ваду предиктивних систем (пор. Allegheny AFST). Прозорий адитивний score, людина в циклі, дані Рівня 1 — лише privacy-preserving.
