# 🕊️ Ластівка

**Проактивний privacy-preserving захист прав дитини** — крос-реєстрова система раннього виявлення порушень прав дитини в Україні.

Жодне порушення не доводить один реєстр — доводить **перетин** даних кількох. Ластівка збирає дитину з силосованих державних реєстрів (не зливаючи персональні дані), виявляє порушення на перетині сигналів, ранжує за терміновістю й віддає кейсворкеру пояснену чергу.

> Демонстрація на **синтетичних** даних, які (1) структурно повторюють **точні схеми реальних реєстрів** (поля з законодавства) і (2) відтворюють **реальну епідеміологію** дитячих даних України (справжні base rates і кореляції). Жодних справжніх персональних даних.

---

## Що нового (фаза 2): реалізм

- **Точні схеми реєстрів** — поля з законів/постанов/офіційних форм (див. `docs/REGISTRY_SCHEMAS.md`), не «на око».
- **Реальна епідеміологія** — base rates і кореляції з дослідження (50 агентів: Держстат, UNICEF, OHCHR, Мінсоц, НСЗУ, МОН, ILO). Більшість дітей **без порушень** (~77% чисті); рідкісне лишається рідкісним; фактори **корельовані** (коморбідність). Параметри — `config/epidemiology.yaml`, провенанс — `docs/genparams.json`.
- **Гетерогенність** — кожна дитина = унікальний сампл латентного контексту → різні життєві шляхи, не клони.

---

## Чотири стовпи
1. **Інтеграція даних** — зібрати життєві події з реєстрів у профіль дитини.
2. **Виявлення ризиків** — тривожні сигнали й критичні відхилення до кризи.
3. **Цілеспрямоване реагування** — ранжований сигнал відповідальному спеціалісту.
4. **Управлінська панель** — моніторинг і підтримка рішень.

## Архітектура

| # | Модуль | Що робить |
|---|---|---|
| 1 | `realmodel` (+ `entity`) | реалістична популяція: демографія → латентні контекст-фактори (корельовані) → УМОВНЕ призначення порушень (P = base × Π множників) → траєкторія в часі |
| 2 | `emitters` + `storage` | кожен реєстр НЕЗАЛЕЖНО пише у власну SQLite-базу (силос) з **точними реальними полями** + шум + `access_level` |
| 3 | `matching` | зібрати дитину з силосів: точно по УНЗР, fuzzy (`rapidfuzz`) по ПІБ+дата де УНЗР нема (id_map — різні реальні назви полів) |
| 4 | `detection` | формули порушень (перетин реєстрів) + change-point/dropout |
| 5 | `scoring` | тріаж: urgency-score → рівень черги T0/T1/T2 з поясненням |
| 6 | `privacy` | PPRL (Bloom-filter/Dice) + PSI-булеан для даних Рівня 1 |
| 7 | `caseload` | розподіл по кейсворкерах: територія + ємність за нормативом + строки 585 + перелив |
| 8 | `validation` | precision/recall vs ground truth (+ MLflow) |
| 9 | `feedback` | захоплення рішень кейсворкера — джерело майбутніх міток (model-ready шов) |

**Метод:** темпоральна агентна симуляція (стиль Synthea): причина = латентний фактор/шок, симптом = крос-реєстровий лог-патерн виникає сам. Детектор бачить лише силоси, не god-view.

## Запуск

```bash
pip install -r requirements.txt

python scripts/run_generation.py -n 5000      # 17 силосованих реєстрів (точні схеми)
python scripts/run_pipeline.py --no-mlflow    # match → detect → score → caseload → validate
uvicorn app.api:app --port 8000               # API → /docs  (фронт ходить сюди)
streamlit run app/dashboard.py                # або вбудований дашборд → :8501

# фронтенд (Next.js)
cd frontend && npm install && npm run dev      # → http://localhost:3000

# model-ready шов (демонстрація конвеєра калібрування)
python scripts/seed_feedback.py               # демо-рішення кейсворкерів (з ground truth)
python scripts/calibrate.py                   # тренує інтерпретовану калібровану модель + MLflow
```
Масштаб: `-n 50000`. Параметри: `config/config.yaml`, `config/epidemiology.yaml`, `config/scoring.yaml`.
Фронт читає API через `NEXT_PUBLIC_API_BASE` (без нього — демо-моки).

## Метрики (5 000 дітей)

- **Реалізм:** ~77% дітей чисті · бідність ~64% · ВПО ~15% · сироти ~0.9% · депортовані ~0.3% · інвалідність ~2.9% — у межах реальних оцінок.
- **Матчинг:** ~96% дітей зібрано з силосів у 1 сутність.
- **Детекція:** precision ~0.94 · recall ~0.84 · F1 ~0.89 (15 типів порушень, vs ground truth).
- **PPRL (без plaintext):** precision ~1.00 · recall ~0.95.

## Розподіл по кейсворкерах (обмежена кількість працівників)
Черга маршрутизується **за місцем проживання дитини** (ССД області/громади), ріжеться за **нормативом ємності** (~10–12 активних кейсів/працівник), рівні T0/T1/T2 = **реальні строки реагування** (ПКМУ №585: оцінка безпеки за 1 добу). Надлишок понад ємність → ескалація + сигнал «потрібно N штатних одиниць». Деталі — `docs/CASEWORKER_PRACTICE.md`, `docs/CASEWORKER_RANKING_DESIGN.md`.

## Model-ready (на майбутні реальні дані)
Зараз ранжує **прозора формула** (на синтетиці навчати модель безглуздо — циклічно). Шов закладено: `feedback` логує рішення кейсворкерів (джерело міток), `scripts/calibrate.py` навчає **інтерпретовану** калібровану модель (логістична, з прозорими коефіцієнтами), `scoring` підхоплює її поруч із формулою. Реальна цінність — лише з реальними підтвердженими випадками (як AFST, але з прозорістю й людиною в циклі).

## Безпека, федеративність, аудит
- **Compute-to-data (федеративна LRA)** — кожен реєстр рахує сигнали **локально** і віддає лише похідний `envelope`, не сирі дані; C1-агрегатор зводить за псевдонімом (HMAC, не УНЗР) і застосовує ті самі правила. Ідентично централі (F1 0.874), стіни (ЕСОЗ/ЄРДР) **push-only**. Код: `lastivka/federated.py`; режим `pipeline.run_pipeline(federated=True)`; візуалізація — сторінка `/federated` + ендпоінти `/federated/stats`, `/federated/{id}`.
- **Аудит «ризик vs хвороба»** — багаторазовий харнес фолт-інжекції (`lastivka/audit/`): confounder-isolation differential («справжній ризик» проти «дитина просто захворіла») + F1-регрес-гейт. Знайшов і виправив FP-конфаундери, при цьому **не заглушив реальний ризик** (запобіжники врятували F3/F4). Звіт: `out/audit/report.md`.
- **Крос-кордон UA↔EE** — privacy-preserving зв'язок з естонськими реєстрами: через кордон перетинає лише результат PPRL-збігу (GDPR гл.V), зміст TERVIS не передається. `lastivka/crossborder.py`.

## Етика
Decision **support**, не decision making: система пріоритезує й **пояснює**, рішення ухвалює спеціаліст. Виявляємо **об'єктивну депривацію прав**, а не «прогнозуємо злочинність батьків» — це обходить головну ваду предиктивних систем (пор. Allegheny AFST). Прозорий адитивний score, людина в циклі, дані Рівня 1 — лише privacy-preserving.
