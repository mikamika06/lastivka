# 🕊️ Ластівка

**Проактивний privacy-preserving захист прав дитини** — крос-реєстрова система раннього виявлення порушень прав дитини в Україні.

Жодне порушення не доводить один реєстр — доводить **перетин** даних кількох. Ластівка збирає дитину з силосованих державних реєстрів (не зливаючи персональні дані), виявляє порушення на перетині сигналів, ранжує за терміновістю й віддає кейсворкеру пояснену чергу.

> Демонстрація на **синтетичних** даних, структурно ідентичних реальним реєстрам Trembita/X-Road. Жодних справжніх персональних даних.

---

## Чотири стовпи

1. **Інтеграція даних** — зібрати життєві події з реєстрів (цив. стан, охорона здоров'я, освіта…) у профіль дитини.
2. **Виявлення ризиків** — відстежити тривожні сигнали й критичні відхилення до того, як вони стануть кризою.
3. **Цілеспрямоване реагування** — надіслати ранжований сигнал відповідальному спеціалісту.
4. **Управлінська панель** — єдина панель моніторингу й підтримки рішень.

---

## Архітектура (8 модулів)

| # | Модуль | Що робить |
|---|---|---|
| 1 | `entity` + `trajectory` | популяція дітей із прихованим станом у часі + шоки (= мітки порушень) |
| 2 | `emitters` + `storage` | кожен реєстр НЕЗАЛЕЖНО пише у власну SQLite-базу (силос) + шум + `access_level` |
| 3 | `matching` | зібрати дитину з силосів: точно по УНЗР, fuzzy (`rapidfuzz`) по ПІБ+дата де УНЗР нема |
| 4 | `detection` | формули порушень (перетин реєстрів) + change-point/dropout |
| 5 | `scoring` | тріаж: urgency-score → рівень черги T0/T1/T2 з поясненням |
| 6 | `privacy` | PPRL (Bloom-filter/Dice) + PSI-булеан для даних Рівня 1 |
| 8 | `validation` | precision/recall vs ground truth (+ MLflow) |

**Метод генерації** — темпоральна агентна симуляція (як Synthea для EHR): причина = прихований шок, симптом = крос-реєстровий лог-патерн виникає сам. Детектор бачить лише силоси, не god-view.

## Запуск

```bash
pip install -r requirements.txt

# 1) згенерувати силосовані реєстри
python scripts/run_generation.py -n 5000

# 2) прогнати пайплайн (match → detect → score → validate)
python scripts/run_pipeline.py --no-mlflow

# 3) дашборд
streamlit run app/dashboard.py        # http://localhost:8501

# (опц.) API
uvicorn app.api:app --port 8000        # http://localhost:8000/docs
```

Масштаб: `-n 50000` (чи 500000) для більшої популяції. Конфіги — `config/config.yaml` (генерація) і `config/scoring.yaml` (ваги тріажу, калібруються).

---

## Поточні метрики (5 000 дітей)

- **Матчинг:** ~96% дітей зібрано з силосів у 1 сутність; fuzzy витягує записи без УНЗР.
- **Детекція:** precision ~0.98 · recall ~0.89 · F1 ~0.93 (vs синтетичний ground truth).
- **PPRL (без plaintext):** precision ~1.00 · recall ~0.95 — приватний матчинг доведено.

---

## Етика

Decision **support**, не decision making: система пріоритезує й **пояснює**, рішення ухвалює спеціаліст. Виявляємо **об'єктивну депривацію прав** (дитина записана в школі чи ні), а не «прогнозуємо злочинність батьків» — це обходить головну ваду предиктивних систем (пор. Allegheny AFST). Прозорий адитивний score, людина в циклі, дані Рівня 1 — лише privacy-preserving.
