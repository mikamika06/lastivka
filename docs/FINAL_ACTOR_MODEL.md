# Ластівка — Фінальна модель акторів, доступу та режимів

> Зведення двох дослідницьких проходів (актори/доступ + воєнні актори/моніторинг/батьківський ризик), прочесаних по всіх документах проєкту та заземлених на чинне право (з адверсарною перевіркою). Це авторитетна модель для реалізації. Дизайн — серйозний, державного рівня (Дія/Мінцифри): без емоджі, без багатослівних пояснень «захищено тим-то».

## 1. Хто фінальний користувач — 8 персон

Кожна персона — окремий рівень авторизації з **окремим landing, окремим nav і окремим скоупом даних у дата-шарі** (не приховуванням у розмітці). Поточний фронт помилково зливав усе в одну роль — це центральний дефект, який усувається.

| Код | Персона | Скоуп даних (максимум легально) | Стіни → що показуємо | Landing |
|---|---|---|---|---|
| **SSD** | Посадова особа ССД (Ф-Г громада / Ф-М місто / Ф-О район) | **Єдина** інтегрована крос-реєстрова роль (Рівень-2), **у межах своєї території**. Прямий: ЄІАС «Діти». На запит/Трембіту: ДРАЦС/ЄДДР/ЄДЕБО/ІСУО/ВПО/Діти-війни/ЦБІ. Відкрито: ЄДРСР/ДРРП. ДН повний. ПФУ — маркер. EE — лише PPRL-сигнал. | ЄРДР і eHealth — **не бачить навіть присутності**: лічильник «N захищених джерел» (без назв/причин) | `/queue` |
| **CNSP** | Фахівець із соц. роботи / ЦНСП (надавач послуг) | **Вузький, case-scoped**: лише власні відкриті випадки (`assigned_to==workerId`), дані сімʼї/кейсу через рольові підсистеми ЄІССС. **Без** інтегрованого крос-реєстру, PSI, особової справи, чужих кейсів | Усе крос-реєстрове/walled — поза доступом (не наглядова роль); walled приходить лише як вихідне процесуальне повідомлення | `/my-queue` |
| **OMBUD** | Уповноважений ВРУ з прав людини (омбудсман) + уповноважені нагляду | **Найширший читач**: per-record PII по всіх ролях і територіях, повні особові справи, доказова база, аудит-лог (ЗУ 776/97-ВР ст.13 п.5 — доступ до інформації з обмеженим доступом + копії; не обмежується за воєнного стану) | Де закон забороняє **зміст** — рівно **alert без причини** (див. §2) | `/oversight` |
| **SUPERVISOR** | Наглядач-аналітик (НСССУ / Мінсоц) | Агрегати/KPI/якість даних/контроль користувачів; персональні — лише за службовою потребою; ЄРДР/СКАЙД — held-released агрегати держателя + придушення малих клітинок | Той самий alert без причини; per-record walled — взагалі немає | `/dashboard` |
| **REGIONAL** | Регіональний рівень (область) | **Лише обласні агрегати/тренди/KPI/географія**, без жодного PII (ПІБ/УНЗР/ДН вирізано в дата-шарі); small-cell suppression | Walled — лише held-released агрегати | `/dashboard` |
| **VERTICAL** | Лікар (МОЗ) · Школа (МОН) · Поліція (МВС) · Мінʼюст · ПФУ | **Лише власний реєстр** + переважно **отримує адресні сигнали** (джерело+отримувач, не браузер). Поліція читає зміст ЄРДР (власна вертикаль) | Чужі вертикалі — лише **вхідні адресні push-сигнали**, не читання | `/inbox` |
| **PARENT** | Батьки (Дія.Підпис) | **Лише власна дитина**: статуси/нагадування/поради | Пригнічені реєстри (кейс/ЄІАС/ЄРДР/ДН/СКАЙД/лінії) — **повна відсутність** (батько може бути фігурантом; заявник конфіденційний) — навіть не alert | `/child` |
| **PUBLIC / ADMIN** | Публіка / Адміністратор | Публіка: знеособлені агрегати + публічно-відкрите (ЄДРСР/ДРРП/ЄДЕБО). Адмін: користувачі/ролі/інтеграції/аудит — **без** кейс-даних | — | `/public` / `/admin` |

**Два працівники, що НЕ контактують (load-bearing, ACTORS §2.5):** **ССД** = владний орган опіки, decision-maker, інтегрований доступ; **ЦНСП/фахівець** = надавач послуг без владних повноважень, case-scoped. Фахівець передає інформацію **вгору** до ССД; вони не рівні й не зливаються в одну роль. Це різні персони з різним landing/nav/скоупом.

**Наглядач ≠ Омбудсман:** омбудсман — максимум легального (per-record PII); наглядач-аналітик — агрегати. Спільна межа: де закон забороняє зміст — alert без причини.

## 2. Правило ALERT-WITHOUT-REASON (content-free + reason-free)

Застосовне **лише до наглядових ролей** (OMBUD, SUPERVISOR) і **тільки на стінах** WALLED={ЄРДР, eHealth, TERVIS}. Рендеримо рівно **одну сіру плитку**: «**Захищений сигнал. Зміст і підстава недоступні за законом.**» — і **нічого більше**.

**Заборонено показувати:** (1) назву реєстру/системи; (2) тип/предмет («розслідування»/«медичне»); (3) причину/підставу («ст.222»/«медтаємниця»); (4) дату, лічильник, presence-булеан «є/нема»; (5) будь-яку метадану для реконструкції.

**Чому без «чому»:** саме існування designated-таємниці досудового розслідування — теж відомості, розкриття яких заборонене (КПК ст.222 — лише з письмового дозволу слідчого/прокурора; ст.387 КК). Система не може ні підтвердити, ні спростувати, ні пояснити.

**Реалізація в дата-шарі, не в JSX:** новий примітив **`walled_alert`** повертає рівно `{alert: true}` (без `present`, `registry`, `reason`, `date`) — окремий від поточного `signal_only` (який витікає `present:bool`+`note`). 
- OMBUD/SUPERVISOR на walled → `{alert:true}`.
- SSD на walled → лічильник `protected_sources` («N захищених джерел», без назв/причин).
- Решта ролей → повна відсутність.

## 3. Два режими продукту

| | РЕЖИМ 1 — ВИЯВЛЕННЯ (нове) | РЕЖИМ 2 — МОНІТОРИНГ (вже-постраждалі) |
|---|---|---|
| Когорта | відкрита популяція | **зараховані** діти: депортовані/повернені, «Діти війни» (ПКМУ 268), сироти/ПБП під опікою, UASC, постраждалі від насильства |
| Тригер | новий сигнал (інтейк/школа/крос-реєстрова розбіжність) | плановий перегляд (~180 днів), віха плану, зміна статусу |
| Питання | «чи є новий ризик — відкрити кейс?» | «чи план реінтеграції/догляду на треку, чи погіршується?» |
| Вихід | ранжована черга → відкрити кейс | прогрес плану + **флаг погіршення** |
| Екрани | `/queue`, `/profile` | `/caseload`, `/my-queue` + трекер плану |

**Запобіжник:** «вже постраждав» (депортована/постраждала/сирота) = **флаг зарахування в моніторинг, НЕ штраф** до нового ризику. Статус зараховує в Режим 2, але не підіймає бал Режиму 1.

## 4. Батьківсько-сімейна вісь ризику (`dimension=parental`)

Ризик дитини враховує **сімейний граф**: дані про батьків, дані лікарів про батьків, освіту, родичів, братів/сестер. Емітується як `Contribution[]` з тегом `dimension="parental"`, що входить у наявну агрегацію `top + 0.2·Σ` (а не лише множником).

| Сигнал | Джерело | Стіна | База (seed) |
|---|---|---|---|
| parent_criminal (насильницький / будь-який) | **НЕ ЄРДР** (ст.222) — push / post-conviction відкрите (Реєстр засуджених, судимість-витяг, ЄДРСР-вирок) | факт-судимості, не зміст провадження | 0.55 / 0.35 |
| parent_addiction / parent_mh | **НЕ eHealth** (медтаємниця) — push/consent булеан; предиктор = **функціональний вплив на догляд**, не діагноз | fact-only floor 0.6 | 0.45 |
| parent_rights_deprived | ЄДРСР (відкрито) — вже `court_deprivation` | спрацьовує й на **сиблінгів** | 0.50 |
| sibling_in_care / sibling_prior_violation | ЄІАС «Діти» через **PSI на C1** | сигнал, не сирі записи; контекст, не детермінізм | 0.40 / 0.45 |
| unemployment / household_churn | ПФУ | **бідність≠ризик**: context-only, ≤0.2 у density | 0.0 standalone |
| new_cohabitant_recent | ЄІАС/ЄІССС (uncertain) | context-корроборація на P1/F1 | 0.0 standalone |
| kinship_care (родичі-опікуни) | ЄІАС «Діти» | **захисний** модифікатор (може **знижувати**) | — |

**Запобіжники:** бідність≠ризик; немає даних=unknown (не штраф); медтаємниця — факт, не зміст; КПК-222 стіна; toxic-trio — контекст, не авто-бал; **батьківське завжди нижче за дитячі тяжкості** (сам по собі → «спостереження», не T0, без корроборації PSI≥2 / віку); без подвійного рахунку; seed-ваги (калібрувати на ground-truth); людина-в-циклі (Комісія з питань захисту прав дитини) обовʼязкова.

**Знахідка (закриває §7 handoff):** зараз **немовля з батьком-кривдником без власного порушення набирає 0 і випадає з черги** (порожній `det` → `raw=0`). Батьківська вісь робить цей превентивний кейс досяжним.

## 5. Воєнні та координаційні актори (контекст, не нові споживачі даних)

- **ОВА/РВА** (обласні/районні військові адміністрації): **ССД — структурний підрозділ ОВА** (Указ 68/2022). ОВА = владна обгортка над ССД, **без нового права на дані**.
- **ВЦА** (військово-цивільні адміністрації) — чинні, аналог на місцях.
- **Комісія з питань захисту прав дитини** — колегіальний орган, що **ухвалює** влаштування/відібрання (Ластівка = підтримка рішень, Комісія = ухвалення).
- **Держслужба України у справах дітей** (ПКМУ 1048/2023) — координація/реінтеграція; **не** володілець ЄІАС «Діти» (це Мінсоц).
- **Нацсоцслужба** — залишковий нагляд/якість.
- **Радник-уповноважений Президента з прав дитини** (Дарʼя Герасимчук, Указ 245/2021) — дорадчий.
- **Координаційний штаб з прав дитини** (ПКМУ 302/2022), **Bring Kids Back UA**, платформа **«Діти війни»** (childrenofwar.gov.ua), **Реєстр депортованих дітей** (Мінʼюст, ПКМУ 1240/2024) — режим моніторингу.
- **Гарячі лінії** 116 111 (Ла Страда) / 1547 / 1545, **НГО** (Save Ukraine, SOS, UNICEF Spilno) — інтейк/реінтеграція, через push/referral, не сирі дані.

## 6. План реалізації (фронтенд + бекенд)

**A. Сегрегація персон (ядро — «всі бачать все» → кожен своє):**
1. `session.ts`: розширити `OpRole {community,regional}` → `AppRole` (ssd/cnsp/ombud/supervisor/regional/doctor/school/police/parent/admin) + `dataScope`, `oblast/hromada`, `workerId` (cnsp), `pii`, `vertical`. **Розділити `community` на `ssd` (територіальний) і `cnsp` (case-scoped).**
2. `nav.tsx`: `NAV_BY_ROLE: Record<AppRole, NavItem[]>`; Sidebar рендерить лише дозволене (чужий nav не потрапляє в DOM).
3. Landing за роллю (login/гард): ssd→/queue, cnsp→/my-queue, ombud→/oversight, supervisor/regional→/dashboard, vertical→/inbox, parent→/child.
4. Дата-шар (`lib/api.ts` + `roles.py`): cnsp=case-scope (assigned), regional/supervisor=без PII+small-cell, ombud=all-PII, vertical=own-registry, parent=own-child+perpetrator-guard, ssd=територія.
5. **`walled_alert`** примітив (§2) + компонент `<WalledAlert/>` (одна плитка, без props причини).
6. Нові екрани: `/oversight` (ombud), `/inbox` (vertical), `/child` (parent), `/audit`.
7. Прибрати емоджі/багатослівні «захищено…» рядки (intake/family/profile).

**B. Батьківська вісь ризику (§4):** `familygraph.py` push-флаги; `detection.parental_contributions`; `scoring` `dimension=parental` + `scoring.yaml` `parental_severity`; тест: немовля+кривдник → T2 (не 0).

**C. Режим моніторингу (§3):** когортний енумератор + трекер плану реінтеграції; статус=зарахування, не штраф.
