# 🎨 Ластівка — бриф для фронтенду (орієнтовний)

Це **орієнтир**, не жорсткий ТЗ. Мета — гарний, серйозний, зрозумілий інтерфейс держрівня для системи захисту прав дитини. Бекенд уже працює (Python: пайплайн + FastAPI + є проста Streamlit-версія як референс UX). Твоя задача — зробити **дуже файний фронтенд** поверх API.

---

## Що це за продукт (одне речення)
Система **раннього виявлення порушень прав дитини**: збирає дитину з розрізнених держреєстрів (не зливаючи персональні дані), знаходить порушення на **перетині сигналів**, ранжує за **терміновістю** і віддає спеціалісту **пояснену чергу** на реагування.

## Вайб / дизайн-напрям
- **GovTech, довіра, спокій.** Не «веселий дашборд» — це про дітей у біді. Чисто, повітряно, читабельно.
- **Українська мова**, доступність (контраст, розмір шрифту), адаптив.
- **Кольори терміновості** наскрізні: 🔴 T0 (червоний) · 🟠 T1 (помаранчевий) · 🟡 T2 (жовтий). Решта — нейтральна (графіт/світлий фон, один акцент).
- Стиль як на нашому концепт-слайді: світлий фон, тонкі неонові акценти, великі цифри-індекси розділів, картки із заокругленнями.

## Екрани (4 стовпи продукту)

### ④ Управлінська панель (головна / огляд)
Для керівника: загальна картина.
- KPI-картки зверху: **T0 / T1 / T2 / Негайних** (великі цифри).
- Графіки: розподіл **за типами порушень** (горизонт. бар), **за регіонами** (топ-10), **рівні черги** (донат).
- Блок **якість моделі**: precision / recall / F1 / % матчингу (як «знак довіри»).

### ③ Черга реагування (ядро продукту)
Ранжований список кейсів — те, з чим працює кейсворкер щодня.
- Кожен кейс = картка: **tier-бейдж**, score, ПІБ (вік), типи порушень, мітка `🚨 НЕГАЙНО` для immediate.
- Фільтри: рівень (T0/T1/T2), «лише негайні», тип порушення, регіон.
- Картка **розкривається в ПОЯСНЕННЯ** (це критично — не чорна скриня):
  - які порушення, з яким внеском у score;
  - **перетин яких реєстрів** це довів;
  - тяжкість + гострота;
  - **фактори вразливості** (вік<6, ВПО, без опікуна…);
  - 🔒 позначка, якщо сигнал з даних **Рівня 1** (ЄРДР/психіатрія) — «отримано як PSI-булеан, повний доступ за ухвалою суду».
- Дія: «взяти в роботу» / «призначити спеціаліста» (поки заглушка — статус кейсу).
- ⚠️ Тон UI: **decision support, не вирок** — система пріоритезує й пояснює, рішення ухвалює людина.

### ① Профіль дитини (інтеграція даних)
Як система «бачить» дитину, зібрану з силосів.
- Шапка: ПІБ, **УНЗР**, дата народження, регіон, **скільки реєстрів інтегровано**.
- Чіпи **силосів**, з яких зібрано (ДРАЦС, eHealth, ЄДЕБО…).
- **Таймлайн подій** з різних реєстрів (переміщення → вихід зі школи → закрита декларація → виклик поліції…).
- **Графік відвідуваності/успішності** з підсвіченою точкою **зламу** (change-point) — візуально показує момент, коли почалось погіршення.

### ② Приватність і якість моделі (довіра)
- **PPRL-блок**: матчинг по зашифрованих відбитках, метрики «без plaintext».
- Таблиця precision/recall **по кожному порушенню** + загальний F1.
- Коротке пояснення приватності (медтаємниця, Рівень-1 через PSI).

## Дані / API (звідки брати)
Бекенд — FastAPI (`app/api.py`), запуск `uvicorn app.api:app --port 8000`, є `/docs` (Swagger).

| Метод | Що повертає (орієнтовно) |
|---|---|
| `GET /queue?tier=&immediate=&limit=` | `{count, items:[{rank, entity_id, pib, age, tier, score, immediate, violations:[], vuln_factors:[]}]}` |
| `GET /entity/{id}` | `{entity_id, unzr, pib, birth_date, registries:[], n_registries, records:{reg:count}}` |
| `GET /metrics` | `{matching, detection:{overall, per_violation}, privacy}` |

> Для повноцінної черги/профілю фронту знадобляться ще **contributions** (пояснення score) і **timeline** (події дитини). Зараз вони лежать у `out/pipeline.db` (таблиця `queue`, поле `contributions`) і в матчингу. Скажи мені — **розширю API** (додам `/queue` з `contributions`, `/entity/{id}/timeline`), щоб ти брав усе одним запитом, а не ліз у БД.

## Технічно (орієнтовно, на твій смак)
- React / Next.js (або Vue) + читати FastAPI. Бібліотеки графіків — Recharts / Visx / ECharts.
- Можна почати з **моків** за схемами вище, а потім підключити реальний API.
- Streamlit-версія (`app/dashboard.py`) — **референс UX** (що і де показувати), не дизайн-еталон. Подивись, як воно поводиться, і зроби краще.

## Словник (щоб не плутатись)
- **УНЗР** — унікальний номер запису (ключ дитини між реєстрами).
- **Силос** — окрема БД реєстру (їх не можна зливати — приватність).
- **Tier T0/T1/T2** — терміновість: сьогодні / цей тиждень / спостереження.
- **immediate** — порушення, що йде в топ без очікування (торгівля, депортація, сексуальне насильство).
- **Рівень доступу 1/2/3** — правовий бар'єр до даних; Рівень-1 = лише як privacy-булеан.
