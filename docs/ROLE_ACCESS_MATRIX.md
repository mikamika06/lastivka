# Ролі × Реєстри × Доступ — хто що бачить і чим користується

> Виведено з `docs/CHALLENGE_REQUIREMENTS.md` (аркуш «Role», 9 ролей) + наших реєстрів (`docs/REGISTRY_SCHEMAS.md`) + моделі рівнів доступу (1 судова ухвала/медтаємниця · 2 ССД через Трембіту · 3 публічно).
> Принцип: **сегрегація даних за роллю** — лікар не бачить суд, школа не бачить медичне, батьки — лише свою дитину, поліція — кримінал/ДН. Крос-реєстрове виявлення робить лише ССД-рівень.

## 0. Три відношення до системи (хто такий «користувач»)
| Тип | Хто | Роль у системі |
|---|---|---|
| **Оператор** | Фахівець ССД (Ф-Г/Ф-М/Ф-О) | єдиний інтегрований крос-реєстровий доступ; бачить профіль + ред-флаги; веде кейс |
| **Вертикаль (джерело+отримувач)** | Лікар (МОЗ), Школа (МОН), Поліція (МВС), Мінʼюст, ПФУ, Мінветеранів | бачить ЛИШЕ свій реєстр; живить дані; отримує адресний сигнал |
| **Кінцевий споживач** | Батьки (додаток ЦШД), Наглядач/Аналітик (НСССУ/Мінсоц), Публіка | своя дитина / агрегати / знеособлене |

## 2. Що кожна роль РОБИТЬ (призначення)

### Фахівець ССД (Ф-Г громада / Ф-М місто / Ф-О область) — ОПЕРАТОР
- **Доступ:** інтегрований крос-реєстровий (Рівень 2), **у межах своєї території** (Ф-Г — громада, Ф-М — місто, Ф-О — область). Рівень-1 (eHealth, ЄРДР) — лише сигнал, повний за ухвалою.
- **Що юзає:** зібраний профіль дитини, чергу ред-флагів/відхилень, контрольні візити, строки реагування (585), сімейні форми влаштування.
- **Це наш головний користувач** — під нього зроблено чергу/тріаж/профіль/caseload.

### Лікар (вертикаль МОЗ)
- **Доступ:** eHealth/ЕСОЗ (повна медкарта, **медтаємниця**), ЦБІ. **НЕ бачить** ЄРДР/суд/соц/освіту.
- **Що юзає:** медичні події, вакцинація, декларація. **Обовʼязок:** повідомити ССД+поліцію про ознаки насильства (≤24 год / ≤3 год у закладі) → тоді сигнал іде в ССД.

### Школа (вертикаль МОН)
- **Доступ:** ЄДЕБО, АІКОМ. **НЕ бачить** медичне/суд/кримінал.
- **Що юзає:** зарахування, відвідуваність, успішність. **Обовʼязок:** повідомити ССД+поліцію на 10-й робочий день відсутності.

### Поліція / ювенальна превенція (вертикаль МВС)
- **Доступ:** ЄРДР, реєстр ДН, ІКС СКАЙД, ЄДДР; ЄДРСР. **НЕ бачить зміст** медичного/освітнього (лише сигнал).
- **Що юзає:** кримінальні провадження, домашнє насильство, профілактичний облік, заборонні приписи.

### Мінʼюст / ПФУ / Мінветеранів (інші вертикалі)
- **Мінʼюст:** ДРАЦС, ЄДРСР, ДРРП. **ПФУ:** реєстр застрахованих, виплати. **Мінветеранів:** ветеранські дані. Кожен — лише свій зріз.

### Батьки (Б — додаток ЦШД)
- **Доступ:** **лише власна дитина** (через Дія.Підпис). Статуси, нагадування, виплати, поради.
- **Що юзає:** family-side застосунок (друга частина Ластівки) — доступ до допомоги/сервісів, екстрена інформація.

### Наглядач/Аналітик (Н-Р НСССУ / Н-Н Мінсоц)
- **Доступ:** агреговані KPI, якість даних, контроль користувачів, інтернатний блок; персональні дані — **лише за службовою потребою**.
- **Що юзає:** моніторинг якості, макро-аналітика, звіти.

### Адміністратор / Публічний
- **Адм:** користувачі/ролі/інтеграції/аудит — **без доступу до кейс-даних**. **Публіка:** лише знеособлені агрегати.

## 3. Як це лягає на наш код (поточний стан)
- **Маємо:** `access_level` (1/2/3) на кожному реєстрі; privacy-шар (Рівень-1 → PSI-сигнал); одну модель кейсворкера + caseload-розподіл за територією.
- **Бракує:** рольового розмежування доступу — зараз профіль показує **все** одному типу користувача. Треба:
  1. **Роль користувача** (з 9 кодів) як параметр; фільтрувати `rows_by_reg` за матрицею вище.
  2. **Вертикальні вікна** (лікар/школа/поліція бачать лише свій реєстр).
  3. **Маршрутизація сигналу** до правильної ролі (детекція на ССД-рівні → флаг → відповідальний фахівець; кримінал → поліція).
  4. **Батьківський режим** (своя дитина, Дія.Підпис).
  5. Скоуп території для Ф-Г/Ф-М/Ф-О (у нас уже є oblast у caseload — розширити).

## 4. Ключовий принцип (для журі й для коду)
**Крос-реєстрове виявлення червоних прапорців — виключно на рівні ССД** (єдина роль з інтегрованим Рівнем-2). Усі інші ролі — вертикальні зрізи або агрегати. Найчутливіше (медичне, кримінал) перетинає межу лише як **сигнал через PSI**, повний доступ — за судовою ухвалою. Це і є юридично коректне розмежування «лікар ≠ суд».
